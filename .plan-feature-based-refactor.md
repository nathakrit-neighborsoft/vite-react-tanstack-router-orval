# Plan: Frontend Feature-Based Refactor

## 1. สถานะปัจจุบัน

โครงสร้างปัจจุบันเป็น **type-based (layered)** — แบ่งตามประเภทไฟล์ ไม่ใช่ feature-based

```
src/
├── api/            client.ts, server.d.ts (types backend), use-eden-query.ts
├── components/ui/  button.tsx, input.tsx (shadcn)
├── lib/            auth-client.ts, utils.ts (cn)
├── routes/         __root.tsx, index.tsx, drone.tsx
├── env.ts          ← สร้างไว้แต่ไม่มีใคร import (dead code)
├── main.tsx        ← ส่ง queryClient เข้า router context แต่ route ไม่มี loader
└── routeTree.gen.ts
```

## 2. ปัญหาที่พบ

| # | ปัญหา | ผลกระทบ |
|---|------|--------|
| P1 | `routes/drone.tsx` รวม **auth form (sign in/up) + drone list + session logic** ไว้ในไฟล์เดียว | ละเมิด SRP, ไม่ feature-based |
| P2 | router รับ `context.queryClient` แต่ไม่มี route ใช้ `loader` prefetch เลย | เสียจุดเด่น Router+Query integration |
| P3 | `main.tsx` ไม่ได้ Register Router type (`declare module`) | `Link`/`useNavigate`/`useParams` ไม่ type-safe |
| P4 | `env.ts` dead code ขณะที่ `api/client.ts`/`auth-client.ts` ใช้ `import.meta.env` ตรงๆ | สองแหล่งความจริง |
| P5 | query key `['drones']` เขียน inline ใน component | พอ feature โตจะกระจัดกระจาย |
| P6 | ไม่มี React Query DevTools | ตรวจสอบ cache ตอน dev ไม่ได้ |

## 3. เป้าหมาย

- ปรับเป็น **feature-based** (auth, drones) เก็บ shared infra ไว้ที่ `lib/`
- routes/ เป็น **thin route definitions** + loader prefetch
- ใช้ TanStack Router+Query loader ให้เป็นประโยชน์
- type-safe navigation (Register Router type)
- กำจัด dead code + รวม env ที่เดียว

## 4. โครงสร้างเป้าหมาย

```
src/
├── app/
│   └── router.ts                 createRouter + export Router type + Register
├── features/
│   ├── auth/
│   │   ├── auth-client.ts        (ย้ายจาก lib/auth-client.ts)
│   │   ├── components/
│   │   │   └── AuthForm.tsx      (แยกออกจาก drone.tsx)
│   │   └── index.ts
│   └── drones/
│       ├── api/
│       │   └── keys.ts           queryKeyFactory (dronesKeys)
│       ├── hooks/
│       │   └── use-drones.ts     useDrones() wrap useEdenQuery
│       ├── components/
│       │   └── DroneList.tsx     list rendering + sign-out
│       └── index.ts
├── lib/                          shared ข้าม feature
│   ├── api/
│   │   ├── client.ts             (ย้ายจาก api/client.ts)
│   │   ├── server.d.ts           (ย้ายจาก api/server.d.ts)
│   │   └── use-eden-query.ts     (ย้ายจาก api/use-eden-query.ts)
│   └── utils.ts
├── components/ui/                shared shadcn (คงที่)
```

**ข้อจำกัดสำคัญ:** TanStack Router file-based routing บังคับให้ route file อยู่ใน `src/routes/` (ตาม `tsr.config.json`) จึงย้าย route file เข้า features/ ไม่ได้ ทางออกคือ routes/ เป็น thin wrapper ที่ import component จาก features/

## 5. ขั้นตอนการทำ

### Phase A — Shared infra (lib/)
1. สร้าง `src/lib/api/` แล้วย้าย `api/client.ts`, `api/server.d.ts`, `api/use-eden-query.ts` เข้าไป
2. ลบ `src/api/` ทิ้ง
3. แก้ import path ทุกจุดที่อ้าง `@/api/client` → `@/lib/api/client`, `@/api/use-eden-query` → `@/lib/api/use-eden-query`

### Phase B — env consolidation
4. ทำให้ `src/env.ts` เป็น single source: เพิ่ม helper/typing ที่จำเป็น
5. แก้ `lib/api/client.ts` และ `features/auth/auth-client.ts` ให้ import จาก `@/env` แทน `import.meta.env` ตรงๆ

### Phase C — auth feature
6. สร้าง `src/features/auth/auth-client.ts` (ย้ายจาก `lib/auth-client.ts`) แก้ baseURL ให้อ่านจาก env
7. สร้าง `src/features/auth/components/AuthForm.tsx` — ย้ายโลจิก sign in/up ออกจาก drone.tsx (รับ props `onSuccess?: () => void` เพื่อ invalidate query หลัง login)
8. สร้าง `src/features/auth/index.ts` barrel

### Phase D — drones feature
9. สร้าง `src/features/drones/api/keys.ts`:
   ```ts
   export const dronesKeys = {
     all: ['drones'] as const,
     lists: () => [...dronesKeys.all, 'list'] as const,
   }
   ```
10. สร้าง `src/features/drones/hooks/use-drones.ts`:
    ```ts
    export function useDrones() {
      return useEdenQuery(dronesKeys.lists(), () => api.api.drone.get())
    }
    ```
11. สร้าง `src/features/drones/components/DroneList.tsx` — รับ `onSignOut`, render list + sign-out button
12. สร้าง barrel `src/features/drones/index.ts`

### Phase E — routes (thin + loader)
13. ปรับ `src/routes/drone.tsx`:
    - ลบ auth form + drone list markup ออก
    - เพิ่ม `loader` ที่เรียก `context.queryClient.ensureQueryData({ queryKey: dronesKeys.lists(), queryFn })`
    - component: gate render ด้วย session (ถ้าไม่มี session แสดง `<AuthForm onSuccess={invalidate} />`, ถ้ามี session แสดง `<DroneList onSignOut={...} />`)
14. `__root.tsx` และ `index.tsx` คงเดิม (อาจแก้ import header)

### Phase F — router setup (main.tsx + app/router.ts)
15. สร้าง `src/app/router.ts`:
    ```ts
    export const router = createRouter({
      routeTree, defaultPreload: 'intent', context: { queryClient },
    })
    declare module '@tanstack/react-router' {
      interface Register { router: typeof router }
    }
    ```
16. ปรับ `src/main.tsx`:
    - import router จาก `@/app/router`
    - เพิ่ม `<ReactQueryDevtools initialIsOpen={false} />` (import.meta.env.DEV gate)

### Phase G — cleanup
17. ลบ `src/api/` (ย้ายหมดแล้ว)
18. ลบ `src/lib/auth-client.ts` (ย้ายแล้ว)
19. regen routeTree (สคริปต์/vite plugin ทำอัตโนมัติตอน dev/build)

## 6. การตัดสินใจสำคัญ / Tradeoff

| หัวข้อ | ทางเลือกที่ใช้ | เหตุผล |
|--------|--------------|--------|
| ตำแหน่ง route files | คงไว้ใน `routes/` (thin) | ข้อจำกัดของ TanStack Router file-based |
| Session gating | render-gate ใน component + invalidate หลัง login (เลิกใช้ `enabled: !!session`) | ให้ loader prefetch ทำงานได้; เมื่อ login สำเร็จ invalidate แล้ว refetch ใหม่ |
| env.ts | ใช้เป็น single source | กำจัด dead code + รวม config |
| DevTools | เปิดเฉพาะ DEV | ไม่กระทบ bundle production |
| query key | `['drones','list']` (lists factory) | ขยายได้เมื่อมี detail/filter |

- ไม่เปลี่ยน styling/Tailwind
