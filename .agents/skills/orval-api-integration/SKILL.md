# Orval API Integration

## Overview

Typed API layer generated by Orval from an OpenAPI spec. Hooks target React Query; fetch runs through a `customInstance` mutator with cookie-based auth.

## Generation

API hooks/types are generated — never edit `src/lib/api/generated/` by hand.

Regenerate from the backend OpenAPI spec:

```bash
bun run gen:types
```

This fetches `http://localhost:9009/api-json` (strips `/api/auth/*`), writes `.orval-spec.json`, then runs Orval per `orval.config.ts`.

Config (`orval.config.ts`): `client: 'react-query'`, `httpClient: 'fetch'`, `mode: 'tags-split'`, mutator at `src/lib/api/mutator.ts`.

## VITE_API_URL

- Origin only — no path. Correct: `http://localhost:3050`. Wrong: `http://localhost:3050/api`.
- `customInstance` prepends `VITE_API_URL` to the endpoint URL for you.
- Empty string (`VITE_API_URL=`) uses same-origin (dev proxy / production nginx).
- Lives in `src/lib/env.ts`, read via `import.meta.env`.

## customInstance (mutator)

`src/lib/api/mutator.ts` — the single fetch entry point Orval routes every call through:

```typescript
export async function customInstance<T>(url: string, options?: RequestInit): Promise<T> {
  const res = await fetch(`${env.VITE_API_URL}${url}`, {
    ...options,
    credentials: 'include',
    headers: { 'Content-Type': 'application/json', ...options?.headers },
  })
  if (!res.ok) {
    const msg = await res.text().catch(() => res.statusText)
    throw new Error(`${res.status} ${res.statusText}: ${msg}`)
  }
  if (res.status === 204) return undefined as T
  return (await res.json()) as T
}
```

- Sends cookies (`credentials: 'include'`) for session auth.
- Throws on non-2xx; returns `undefined` for 204.

## Auth

- Cookie-based sessions. `credentials: 'include'` is baked into `customInstance`.
- Auth client is `better-auth/react` via `src/features/auth/auth-client.ts`.
- Auth form component: `src/features/auth/components/AuthForm.tsx`.

## Generated hooks

`src/lib/api/generated/<tag>/<tag>.ts` exports query + mutation hooks per endpoint:

- Query: `useGetDrones()`, `useGetDrone(id)`
- Mutation: `useCreateDrone()`, `useUpdateDrone()`, `useDeleteDrone()`
- Helpers: `getGetDronesQueryKey(params)`, `getCreateDroneMutationOptions(...)`

Feature wrappers in `src/features/<feature>/hooks/` adapt generated hooks:

```typescript
import { useGetDrones } from '@/lib/api/generated/drones/drones'
import { dronesKeys } from '../api/keys'

export function useDrones() {
  const dronesQuery = useGetDrones()
  return { ...dronesQuery, data: dronesQuery.data?.data }
}
```

## Query keys

Keys live in each feature's `api/keys.ts`. Drone IDs are `uuid: string`:

```typescript
export const dronesKeys = {
  all: ['drones'] as const,
  lists: () => [...dronesKeys.all, 'list'] as const,
  details: () => [...dronesKeys.all, 'detail'] as const,
  detail: (uuid: string) => [...dronesKeys.details(), uuid] as const,
}
```

## Data invalidation

After mutations, always invalidate:

```typescript
qc.invalidateQueries({ queryKey: dronesKeys.all })
```
