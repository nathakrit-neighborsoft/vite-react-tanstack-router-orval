# Project Agent Instructions

## Project

Drone management frontend client. Connects to Elysia (Eden) backend via typed Treaty RPC.

## Stack

- Bun 1.3 (package manager + TS runner)
- Vite 8 + React 19
- TypeScript 5.9
- TanStack Router 1.95 (file-based, auto-code-splitting)
- TanStack Query 5 (server state, cache, mutations)
- `@elysiajs/eden` (typed RPC client)
- `better-auth/react` (auth client state)
- Tailwind CSS v4 + Radix UI primitives (shadcn-style)
- Vitest + Testing Library (tests)
- oxlint (linter) + Prettier (formatter)

## Source layout

```
src/
  app/              Router setup + type registration
  components/ui/    Reusable UI primitives
  features/auth/    Better Auth client + AuthForm component
  features/drones/  Drone API hooks, mutations, CRUD components
  lib/api/          Eden Treaty client, helpers, server types
  routes/           Thin route files (TanStack Router file-based)
  env.ts            Single env source
```

## Hard rules

- **Use Bun** for all scripts (`bun run dev`, `bun run build`, `bun run test`, etc.).
- Never edit `src/routeTree.gen.ts` — it is auto-generated by TanStack Router plugin.
- Never edit `src/lib/api/server.d.ts` directly — regenerate via `bun run gen:types`.
- Keep route files (`src/routes/`) thin; import features from `src/features/*`.
- Do not refactor beyond what the task requires. Do not add new dependencies unless asked.
- Do not commit or push unless explicitly requested.

## API/Auth rules

- `VITE_API_URL` must be **origin only** (e.g. `http://localhost:3050`). Never append `/api` — Eden Treaty appends route paths itself.
- Empty `VITE_API_URL` = use same-origin Vite/nginx proxy (default).
- Vite dev proxy: `/api/*` → `http://localhost:3050/*`, `/auth/*` → `http://localhost:3050/api/auth/*`.
- All Eden Treaty API calls use `credentials: 'include'` for cookie-based auth.
- Use `handleEdenResponse` or `useEdenQuery` helpers from `@/lib/api/` for typed API responses.
- After drone mutations, invalidate `dronesKeys.all` to refetch.

## Routing and data rules

- Register router type in `src/app/router.ts` so `Link`/`useNavigate`/`useParams` are type-safe.
- Query keys live in feature `api/keys.ts` (e.g. `dronesKeys`).
- Use React Query hooks from `@/lib/api/use-eden-query` as a wrapper for TanStack Query.
- Route loaders can use `context.queryClient.ensureQueryData` for prefetching.

## Verification

- Run `bun run lint` before finishing.
- Run `bun run typecheck` after any TypeScript changes.
- Run `bun run build` for a full production build check.
- Run `bun run test` if touching feature code.
- Run `bun run format:check` for formatting.

## What not to do

- Do not edit generated files (`routeTree.gen.ts`, `server.d.ts`).
- Do not add npm/yarn/pnpm lockfiles.
- Do not refactor components/ui/ unless specifically asked.
- Do not inline VITE_API_URL with `/api`.
- Do not skip running `bun run gen:types` when the backend contract changes.
